# GitHub Pages 部署指南

本指南将详细说明如何将项目发布到 GitHub Pages，让全世界都能访问你的网页。

## 前提条件

- 有一个 GitHub 账号（如果没有，请先到 https://github.com 注册）
- 已安装 Git（如果没有，请到 https://git-scm.com 下载安装）

## 第一步：初始化 Git 仓库

1. 打开终端（Terminal），进入项目目录：
   ```bash
   cd /Users/hou/Desktop/boardgame/rule_git
   ```

2. 初始化 Git 仓库：
   ```bash
   git init
   ```

3. 添加所有文件到 Git：
   ```bash
   git add .
   ```

4. 提交文件：
   ```bash
   git commit -m "初始提交：随机选择器项目"
   ```

## 第二步：在 GitHub 上创建仓库

1. 登录 GitHub 网站（https://github.com）

2. 点击右上角的 **"+"** 按钮，选择 **"New repository"**（新建仓库）

3. 填写仓库信息：
   - **Repository name**（仓库名称）：例如 `random-selector` 或 `boardgame-selector`（可以自己起名）
   - **Description**（描述）：可选，例如"随机选择器网页应用"
   - **Public**（公开）：选择 Public（公开），这样 GitHub Pages 才能免费使用
   - **不要勾选** "Initialize this repository with a README"（不要初始化 README）

4. 点击 **"Create repository"**（创建仓库）

5. 创建后，GitHub 会显示一个页面，上面有仓库地址，类似：
   ```
   https://github.com/你的用户名/仓库名.git
   ```
   复制这个地址，下一步会用到。

## 第三步：将代码推送到 GitHub

回到终端，执行以下命令（将 `你的用户名` 和 `仓库名` 替换为实际值）：

```bash
# 添加远程仓库地址
git remote add origin https://github.com/你的用户名/仓库名.git

# 将代码推送到 GitHub
git branch -M main
git push -u origin main
```

**注意**：第一次推送时，GitHub 会要求你输入用户名和密码（或 Personal Access Token）。

- 如果要求输入密码，你需要使用 **Personal Access Token**（不是 GitHub 密码）
- 如何获取 Token：
  1. 登录 GitHub
  2. 点击右上角头像 → **Settings**（设置）
  3. 左侧菜单选择 **Developer settings**
  4. 选择 **Personal access tokens** → **Tokens (classic)**
  5. 点击 **Generate new token** → **Generate new token (classic)**
  6. 填写 Note（备注），勾选 `repo` 权限
  7. 点击 **Generate token**
  8. **复制生成的 token**（只显示一次，请保存好）
  9. 推送时，用户名填你的 GitHub 用户名，密码填这个 token

## 第四步：启用 GitHub Pages

1. 在 GitHub 仓库页面，点击上方的 **"Settings"**（设置）标签

2. 在左侧菜单中找到 **"Pages"**（页面）选项，点击进入

3. 在 **"Source"**（来源）部分：
   - 选择 **"GitHub Actions"**（不是 Deploy from a branch）
   - 如果看到提示，点击 **"Configure"** 或直接保存

4. 等待几秒钟，GitHub Actions 会自动运行部署

## 第五步：查看部署状态和访问地址

1. 在仓库页面，点击上方的 **"Actions"**（操作）标签

2. 你会看到部署任务正在运行（显示黄色圆点）或已完成（显示绿色对勾）

3. 部署完成后（通常需要 1-2 分钟），回到 **"Settings"** → **"Pages"**

4. 你会看到你的网站地址，格式类似：
   ```
   https://你的用户名.github.io/仓库名/
   ```

5. 点击这个地址，就能访问你的网页了！

## 后续更新

以后如果需要更新数据或代码：

1. 修改 `data/` 目录下的 txt 文件
2. 运行 `python3 generate.py` 重新生成 HTML
3. 提交并推送更改：
   ```bash
   git add .
   git commit -m "更新数据"
   git push
   ```
4. GitHub Actions 会自动重新部署，几分钟后网站就会更新

## 常见问题

### Q: 部署失败怎么办？
A: 检查 `.github/workflows/deploy.yml` 文件是否存在，确保文件格式正确。

### Q: 网站显示 404？
A: 等待几分钟让部署完成，或者检查 Actions 标签页看是否有错误。

### Q: 如何修改网站地址？
A: 网站地址是 `https://你的用户名.github.io/仓库名/`，如果要改变，需要修改仓库名称。

### Q: 如何删除网站？
A: 在 Settings → Pages 中，将 Source 改为 "None" 即可。

## 需要帮助？

如果遇到问题，可以：
- 查看 GitHub Actions 的日志（点击 Actions 标签页中的任务）
- 检查终端是否有错误信息
- 确保所有文件都已正确提交到仓库
